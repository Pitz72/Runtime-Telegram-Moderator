# Changelog - v0.1.0

## [0.1.0] - 2026-04-08

### ✨ Evoluzione Infrastrutturale (Consolidamento)

Questa versione segna un salto qualitativo nell'architettura del progetto, integrando pratiche DevOps e QA di livello professionale.

- **Dockerization (Backend)**:
    - Creato `Dockerfile` ottimizzato per lo sviluppo con hot-reload.
    - Implementato `docker-compose.yml` per la gestione orchestrata dei servizi (Backend, Database).
    - Persistenza del database SQLite tramite volumi Docker.
    - Migrazioni Prisma automatiche all'avvio del container.
- **Framework di Testing (Vitest)**:
    - Setup di **Vitest** come motore di unit e integration testing.
    - Configurazione di **Supertest** per la validazione automatica delle rotte API.
    - Creata una suite di test iniziale per `BotManager` e `API` con copertura di mock per Prisma e grammY.
- **Modernizzazione Sviluppo**:
    - Passaggio a `tsx` (TypeScript Execute) per una velocità di esecuzione e watch mode superiore rispetto a `ts-node`.
    - Aggiunta di script npm per test, coverage e watch mode.
- **Linee Guida CLAUDE.md**:
    - Definizione di standard rigorosi per lo sviluppo, l'uso di Docker e le pratiche di coding per garantire la coerenza tra team e AI.

### 🛠️ Dettagli Tecnici
- Aggiornate le dipendenze in `package.json` inclusi `better-sqlite3` per performance ottimizzate.
- Rifattorizzato `backend/src/index.ts` per supportare il testing tramite esportazione dell'app.
- Creati placeholder per la futura integrazione del frontend.
