# Changelog - v0.0.2

## [0.0.2] - 2026-03-14

### ✨ Nuove Funzionalità
- **BotManager Engine**: Implementata la classe `BotManager` per l'orchestrazione dinamica della flotta di bot.
    - Supporto per `startBot`, `stopBot` e `initAllBots`.
    - Gestione delle istanze in una `Map` per controllo real-time.
- **Integrazione grammY**: 
    - Aggiunto comando di test `/ping` per verificare la reattività dei bot.
    - Implementato `bot.catch` globale per prevenire il crash del server in caso di errori API Telegram.
- **REST API (Express)**: Esposti gli endpoint per il controllo della flotta:
    - `GET /api/bots`: Elenco bot e stato.
    - `POST /api/bots`: Registrazione nuovo bot (Token BotFather).
    - `POST /api/bots/:id/start`: Avvio polling per un bot specifico.
    - `POST /api/bots/:id/stop`: Arresto polling per un bot specifico.
- **Persistenza Stato**: Sincronizzazione automatica tra memoria (`Map`) e database (`prisma.bot.update`) per il campo `isRunning`.
- **Auto-Boot**: All'avvio del server, il sistema riavvia automaticamente tutti i bot che erano attivi prima dello spegnimento.

### 🐞 Bug Fixes & Sfide Tecniche
- **Prisma 7 Compatibility**: Risolto errore di validazione dello schema. In Prisma 7, la `url` del datasource deve essere definita in `prisma.config.ts` tramite `@prisma/config` e non direttamente in `schema.prisma`.
- **ESM & TypeScript**: Abilitato il supporto nativo ai moduli ES aggiungendo `"type": "module"` in `package.json`. Questo ha risolto i conflitti di risoluzione dei moduli con il target `NodeNext`.
- **Express 5 Type Safety**: Corretto il tipo di `req.params` negli endpoint dinamici. In Express 5, i parametri possono essere array di stringhe; è stato necessario un casting esplicito a `string` per l'uso con `parseInt`.
- **IDE Ghost Errors**: Gestita la persistenza dell'errore "Missing PrismaClient" tramite rigenerazione forzata del client e verifica manuale con `tsc` (risultata pulita).

### 🧪 Istruzioni di Test (API & Telegram)

Per testare il corretto funzionamento del server e della flotta di bot:

1.  **Avvio del Server**:
    Assicurati di essere nella cartella `backend/` e lancia il comando:
    ```bash
    npm run dev
    ```

2.  **Registrazione di un Nuovo Bot**:
    Esegui una richiesta `POST /api/bots` (utilizzando Postman, Bruno o cURL):
    ```bash
    curl -X POST http://localhost:3000/api/bots \
         -H "Content-Type: application/json" \
         -d '{ "name": "MioBotTest", "token": "IL_TUO_TOKEN_QUI" }'
    ```

3.  **Avvio del Bot**:
    Utilizza l'ID restituito dalla chiamata precedente (solitamente `1` se è il primo bot) per avviarlo:
    ```bash
    curl -X POST http://localhost:3000/api/bots/1/start
    ```

4.  **Verifica su Telegram**:
    Apri la chat su Telegram con il bot che hai appena configurato e digita il comando:
    `/ping`.
    Se tutto funziona correttamente, il bot risponderà: **"Pong da Runtime Moderator!"**.
